# 补码原码反码

## 原码转补码

按位取反再加1

## 补码转原码

1符号位不变

2数值位取反加1

数值位为多少原码就是多少

## 补码特例

8位数时1000 0000表示多少？

按照上面的转换规则 1000 0000的原码就是0  但不是  这是个特例  这个作为-128

| 原码      | 补码      | 数值（不是原码） |
| --------- | --------- | ---------------- |
| 0000 0000 | 0000 0000 | 0                |
| 1000 0001 | 1111 1111 | -1               |
| 1000 0010 | 1111 1110 | -2               |
| 1000 0011 | 1111 1101 | -3               |
| 1000 0100 | 1111 1100 | -4               |
| 1000 0101 | 1111 1011 | -5               |
| 1000 0110 | 1111 1010 | -6               |
| 1000 0111 | 1111 1001 | -7               |
| 1000 1000 | 1111 1000 | -8               |
| 1000 1001 | 1111 0111 | -9               |
| ......    | ......    | .....            |
| 1111 1110 | 1000 0010 | -126             |
| 1111 1111 | 1000 0001 | -127             |
| NULL      | 1000 0000 | -128             |
