# 关于子网掩码和网关

https://www.cnblogs.com/Newlife666/p/19324922)

**网关（Gateway）** 是连接两个不同网络的设备或地址，用于实现跨网段通信。常见的网关设备包括路由器、三层交换机、防火墙等。**默认网关** 是主机在没有更具体路由时发送数据的出口地址，例如家庭网络中路由器的 LAN IP。

**子网掩码（Subnet Mask）** 用于区分 IP 地址中的 **网络号** 和 **主机号**，判断两台设备是否处于同一子网。它由 32 位二进制组成，前面连续的 *1* 表示网络部分，后面连续的 *0* 表示主机部分，例如 *255.255.255.0* 对应二进制 *11111111.11111111.11111111.00000000*。

**判断是否同一子网的原理**

1. 将两台设备的 IP 地址与子网掩码分别转换为二进制。
2. 对 IP 与掩码执行 **按位与运算**（AND）。
3. 若结果（网络地址）相同，则在同一子网，可直接通信；否则需通过网关转发。

**示例**

IP1: 192.168.1.10 -> 11000000.10101000.00000001.00001010

IP2: 192.168.1.200 -> 11000000.10101000.00000001.11001000

掩码: 255.255.255.0 -> 11111111.11111111.11111111.00000000

IP1 & 掩码 = 192.168.1.0

IP2 & 掩码 = 192.168.1.0 → 同一子网

**网关的作用场景**

- **同一子网通信**：无需网关，设备直接通过交换机或集线器通信。
- **跨网段通信**：数据包先发往本地网关，由网关根据路由表转发到目标网络。
- **互联网访问**：局域网设备通过默认网关（路由器）访问外网，路由器可进行 NAT 转换。

**CIDR 表示法** *192.168.0.0/24* 表示子网掩码有 24 位 *1*，即 *255.255.255.0*。 */23* 表示 *255.255.254.0*，可容纳 510 台主机，适合较大局域网。

**配置建议**

- 确保同一子网内设备的 **子网掩码一致**，IP 不重复。

- 默认网关应设置为所在子网出口设备的 IP。

- 大型网络可通过调整掩码或划分 VLAN 提高管理效率。
